# 캔들런 (Candle Run) · v0.15

페이북 투자서비스 활성화를 위한 **실전 캔들 차트 트레이딩 시뮬레이션** 웹게임.
실제 국내주식 과거 데이터와 랜덤 생성 시나리오로 블라인드 매매를 체험하고, 성과를 복기하며 투자 감각을 익힙니다.

## v0.15 변경점 (지속 겜머니 & 수익률 랭킹)

> 계획서: [docs/PLAN_v0.15_SCORING.md](docs/PLAN_v0.15_SCORING.md)

- **페이북겜머니 = 지속 자산(리셋 없음)**: 매 사이클 100만 리셋을 없애고, 게임마다 복리로 **계속 누적/변동**되는 "내 자산"으로 전환.
- **일간 랭킹 = 24시간 누적수익률(%)**: 잔액이 아니라 **사이클 시작(전날 종료) 겜머니 대비 수익률**(= 그날 판들의 ∏(1+rᵢ)−1)로 순위. 자산이 많다고 유리하지 않아 **공정**.
- **상금**: 매 사이클(새벽 6시) 마감 시 **오늘 수익률 상위 3인** 페이북머니 3·2·1만원(현행 유지).
- **파산 하한 보장**: 겜머니는 `MIN_WALLET(1만)` 밑으로 내려가지 않고, 개장 시 하한 이하면 **최소 시드(100만)로 리필**.
- 홈·결과·랭킹 표기를 **원(₩) → 수익률(%)** 중심으로 정리(겜머니 자산은 별도 표시). 영속키 `candlerun-v0.15`.

## v0.14 변경점 (초보자 친화 UX & AI 비서 "캔들이")

> 계획서: [docs/PLAN_v0.14_UX.md](docs/PLAN_v0.14_UX.md)

- **AI 비서 마스코트 "캔들이"**: 촛불 의인화 캐릭터(이모지 기반)가 상황별 표정 + 짧은 한마디로 코칭. 말투는 다정한 **반말 혼용**.
- **매턴 팁·튜토리얼 말풍선화**: 초보자 매턴 팁과 튜토리얼 코치마크를 캔들이 말풍선(표정 포함)으로, 크고 짧게.
- **결과 코칭 리뉴얼**: 좋았던 점 / 주의할 점 / 최선의 한 수를 캔들이가 **짧고 굵게** 전달(핵심 근거 유지).
- **결과 차트 확대·마커 강조**: 리빌 차트 확대(200→300), **내 매수/청산 마커**와 **최선의 한 수 구간(음영·라벨)**을 크게 표시 + 범례.
- **'한 판 더' 자동 팝업**: 결과 도달 시 캔들이가 묻는 팝업(**예/아니오/닫기**). 무료판이면 바로 다음 판, 소진 시 광고. **아니오=홈**, 닫기=결과 유지. 결과 하단 **[홈]·[랭킹]** 복귀 버튼 상시.
- **랭킹 진입 정리**: 별도 랭킹 버튼 축소 → **상단 겜머니 pill·결과 '오늘 N위'를 탭하면 랭킹**으로 이동.
- 코어 규칙(점수=겜머니·사이클·상금)은 그대로, **UX·톤·전달 방식** 중심 리뉴얼.

## v0.13 변경점 (일일 승부 사이클 & 입상 보상)

경쟁형 리텐션 루프로 재편: **매일 100만원으로 시작 → 새벽 6시 결산 → 상위 3인 입상**.

- **매일 100만원 시작**: 매 사이클 시작 시 페이북겜머니를 **100만원으로 리셋**. 손익이 겜머니에 **계속 누적**되어 그 잔액이 곧 **랭킹 점수**.
- **일일 사이클(새벽 7시 개장 → 다음날 새벽 6시 마감)**: 매일 **06:00 결산**해 겜머니 상위 3인을 선정, **1시간 뒤 07:00에 새 라운드 개장**. 6~7시는 결산 시간으로 플레이 잠금(홈에 "🏁 결산 중 · 입상자 발표" + 개장 카운트다운).
- **입상 보상**: 결산 시 겜머니 **1·2·3등에게 페이북머니 30,000 · 20,000 · 10,000원**(프로토타입 표기, 실제 지급 미연동). 랭킹·결과 화면에 상금 뱃지 노출.
- **하루 2판 + 광고 이어하기**: 사이클당 무료 **2판**, 이후 **"광고 보고 한 판 더"**로 본게임 이어하기(모든 판이 겜머니=점수에 누적).
- **기본값 = 초보자 · 30턴**: 첫 진입 난이도를 **초보자**(튜토리얼·매턴 팁 ON), 플레이 봉 수 기본을 **30턴**으로 설정.
- 일간 랭킹 점수를 "오늘 획득 손익 합" → **"현재 겜머니 잔액"**으로 변경(100만 기준 누적). 자정 리셋 → **사이클(새벽 6시 마감) 기준**으로 타이머 재편.

## v0.12 변경점 (게이미피케이션 & 초보자 온보딩)

- **연습게임 제거 → 본게임 이어하기**: 별도 연습 모드를 없애고, **오늘의 종목 무료 2판**을 마치면 **"광고 보고 한 판 더"**로 1판을 **본게임(랭킹 반영)**으로 이어서 플레이. 홈·결과 화면 모두 동일 흐름.
- **초보자 튜토리얼(말풍선 코치마크)**: 난이도 `초보자`에서 게임 시작 시 SIGNAL 읽기 → 차트만 보고 판단 → 매수·공매도·다음턴 순서로 **물풍선 형태 3단계 가이드**를 오버레이(건너뛰기 가능, 새 게임마다 노출).
- **매턴 팁(2초 팝업)**: 초보자 한정, 매 턴 진입 시 상황별 짧은 가이드가 차트 하단에 **2초간 떴다가 사라지는** 눈에 띄는 팝업으로 노출.
- **게임 길이 단축**: 플레이 봉 수 옵션을 30/50/80 → **20/30/50턴**으로 축소(호흡 짧게).
- **ETF/ETN 제외**: 레버리지·인버스·선물·커버드콜 등 **비(非)일반 단일종목을 시나리오 풀에서 제외**(빌드 단계에서 필터, 실데이터 261종목).
- **설정 = 기어(⚙) 아이콘**: 지표·난이도·턴 설정을 메인 **우측 상단 기어 아이콘**으로 이동, 하단 버튼 지양. 설정창은 실행 없이 구성만 담당.
- **빵빠레 폭죽 연출**: 수익으로 마무리하면 결과 화면에 **CSS 폭죽 애니메이션**으로 축하.
- **게이미피케이션 강화(교육효과 유지)**: 튜토리얼·매턴 팁·폭죽으로 몰입을 높이되, SIGNAL·지표 해설·복기(정답 공개, 최선의 한 수)는 그대로 유지.

## v0.4 주요 변경점

- **데이터 ↔ 앱 분리**: 시나리오를 앱에서 떼어내 `app/public/data/`(매니페스트 + 개별 파일)로 관리. 라운드마다 개별 시나리오만 지연 로드 → 초기 로드 3.7MB → **JS 60KB(gzip)** 수준으로 경량화.
- **KOSPI / KOSDAQ 구분**: 시장을 태그·표시하고, 설정에서 시장을 선택할 수 있습니다.
- **데이터 증량 + 최근(2020년+) 데이터**: 실데이터 300 + 랜덤 생성 270 = **총 570개 시나리오**(KOSPI 420 / KOSDAQ 150, 2020년 이후 330개).
- **반응형 동적 UI**: 단말기 화면 크기에 맞춰 차트·레이아웃이 유동적으로 리사이즈(모바일↔데스크톱).
- **명확한 플로우**: `설정 → 실행 → 성과분석 → 지속/종료`.
- **React + TypeScript + Vite**로 재구축(모듈화·타입 안정성·빌드 파이프라인).
- **신규 요소**: 투자 성향 진단, 전환 CTA(데모 훅), 결과 공유, 페이북머니·전적 영속화, 공매도(고급 난이도) 실연결.

## 실행 방법

```bash
cd app
npm install
npm run dev       # 개발 서버 (http://localhost:5173)
# 또는
npm run build     # 프로덕션 빌드 → app/dist
npm run preview   # 빌드 결과 미리보기 (http://localhost:4173)
```

> 데이터는 `fetch`로 로드되므로 **정적 서버로 서빙**해야 합니다(`file://` 직접 열기는 불가). 루트 `index.html`은 `app/dist`로 리다이렉트합니다.

## 구조

```
app/                 # 서비스 앱 (React + TS + Vite)
  src/
    game/            # 게임 엔진: 지표/시그널/엔진/시나리오 로더/성향진단
    components/      # Chart(반응형 캔버스) · 화면(Setup/Play/Result)
    store/           # zustand 스토어 (플로우·지갑·통계·영속화)
  public/data/       # 분리된 데이터: manifest.json + scenarios/*.json
  dist/              # 빌드 산출물
scripts/build_data.py # 데이터 파이프라인(실데이터 추출 + 랜덤 생성 → 분리 저장)
legacy/index-v0.3.html # 이전 프로토타입(단일 파일)
docs/PRD.md          # 제품 요구사항 정의서
```

## 데이터 재생성

```bash
python3 scripts/build_data.py          # app/public/data 갱신
cd app && npm run build                # dist에 반영
```

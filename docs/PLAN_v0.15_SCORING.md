# 캔들런 v0.15 개발 계획서 — 지속 겜머니 & 수익률 랭킹

> **상태: v0.15로 구현 완료.** 아래 확정 결정대로 반영되었습니다.
> **요청 요지**: ① 페이북겜머니는 **리셋 없이 지속** 변동(내 자산처럼 계속 누적) ② 랭킹 점수는 겜머니 잔액이 아니라 **24시간 누적수익률**(그날 곱해진 배수, 또는 전날 종료 겜머니 대비 금일 종료 겜머니 변화).
>
> **확정 결정**: ① 랭킹 지표 = **24h 누적수익률(%)** · ② 경쟁 주기 = **일간(24h)만** · ③ 파산 처리 = **하한 보장(개장 시 최소 시드 리필)** · ④ **바로 구현**.

---

## 1. 핵심 변경 요약

| 항목 | 현행(v0.14) | 제안(v0.15) |
|---|---|---|
| 페이북겜머니 | 매 사이클 **100만으로 리셋** | **리셋 없이 지속 누적**(복리로 계속 변동) |
| 랭킹 점수(일간) | 사이클 **겜머니 잔액**(원) | **24시간 누적수익률**(%) |
| 상금 기준 | 겜머니 잔액 상위 3인 | **24h 수익률** 상위 3인 |
| 공정성 | 같은 100만 출발이라 공정 | 잔액은 누적돼 불공정 → **수익률로 랭킹**해 공정 유지 |

핵심 아이디어: **자산(겜머니)은 계속 쌓이되, 매일의 "경쟁"은 그날의 수익률(%)로 겨룬다.**
→ 오래 한 사람이 잔액이 많다고 매일 유리하지 않음(공정). 자산은 성장의 재미로 별도 유지.

## 2. 모델 정의

### 2.1 자산(겜머니) — 지속
- 신규 유저 시작 `START_MONEY = 100만`.
- 매 게임: 100% 투자 → `W ← max(FLOOR, W × (1 + r))`. **사이클 경계에서 리셋하지 않음.**
- 헤더/홈에 표시되는 "페이북겜머니"는 이 **지속 자산 W**.

### 2.2 사이클(24h) 누적수익률 — 랭킹 점수
- 사이클 개장(현행 새벽 7시) 시점의 자산을 스냅샷: `B_c = W(개장 시각)` = **전날 종료 겜머니**.
- 사이클 중 자산은 복리로 변동.
- **24h 누적수익률** `R_c = W_현재 / B_c − 1`.
  - 이는 그날 각 판 수익률의 곱과 동치: `R_c = ∏(1 + rᵢ) − 1`.
  - 판을 안 하면 `R_c = 0%`.
- **일간 랭킹 = R_c(%)**. 사이클 마감(현행 새벽 6시) 시 R_c 상위 3인에게 페이북머니 3·2·1만.

### 2.3 (옵션) 주간 랭킹
- 주 시작 스냅샷 `B_w`, **주간 누적수익률** `R_w = W / B_w − 1` (= 7일 일수익률 복리).
- 채택 시 일간(24h)과 병행. → "확정 필요 ②".

## 3. 화면/문구 영향

- **헤더·홈 겜머니**: "지속 자산"으로 계속 커지거나 줄어듦(리셋 표기 제거).
- **홈 데일리 카드**: "현재 겜머니(자산)" + **"오늘의 수익률 +x.x%"**(랭킹 점수) 병기, 순위 칩.
- **랭킹 화면**: 일간 점수를 **원(₩) → 수익률(%)**로 표기. 상금 스트립 유지(수익률 상위 3인).
- **결과 화면**: 리워드 pill을 "겜머니 ₩… · 오늘 수익률 +x.x% · N위"로.
- **캔들이 문구**: "오늘 수익률 +x%! 상위권 도전 🔥" 등 수익률 중심으로.

- **① 랭킹 지표 = (A) 24h 누적수익률(%)** — 자산 규모와 무관, 공정.
- **② 경쟁 주기 = (A) 일간(24h)만** — 현행 사이클 유지, 지표만 잔액→수익률. (주간은 추후 옵션)
- **③ 파산 처리 = (A) 하한 보장** — `MIN_WALLET=10,000` 이하로 안 내려가고, 사이클 개장 시 하한 이하면 `START_MONEY(100만)`로 리필.
- **④ 상금** = 일간 수익률 상위 3인(현행 시각·금액 3·2·1만 유지).

## 6. 리스크·고려사항

- **공정성**: 지속 자산으로 잔액 랭킹은 불공정 → **수익률 랭킹**이 핵심 전제. (반드시 함께 가야 함)
- **파산/무한성장**: 100% 복리 지속 → 자산이 0 수렴 또는 과대. FLOOR·상한/표기 정책 필요(③).
- **마이그레이션**: 기존 유저 겜머니(사이클 리셋 기준)를 지속 자산으로 승계. 최초 로드 시 스냅샷 세팅.
- **교육효과**: 수익률(%) 노출은 오히려 학습 지표로 자연스러움(현행 유지·강화).

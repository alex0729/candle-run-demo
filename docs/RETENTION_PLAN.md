# 캔들런 리텐션 루프 작업계획서 (v0.11)

> **KPI**: 재방문을 통한 잠재고객풀 확보 — "매일 돌아올 이유" 루프
> **대원칙**: 강제·강매·퍼널 배제, 초대형(invitational) 톤. 투자/광고 노출은 *안 눌러도 흐름에 지장 없는* 수준.
> **확정 결정**: ① 기존 "3판마다 광고" 게이트 **유지**(오늘의 종목과 병행) ② 랭킹 = **일간(점수)+월간(방문횟수)**, 주간(페이북겜머니) 제거 ③ 일간 점수 = **오늘 획득 페이북겜머니(손익 합)**

## 1. 현재 코드 감사

| 요청 | 현재 | 판정 |
|---|---|---|
| 오늘의 종목(날짜 시드) | 시나리오 `Math.random` 랜덤 | ❌ 신규 |
| 연속 출석 streak | 없음(`adDay`만 광고용) | ❌ 신규 |
| 일간 랭킹+자정 타이머 | 랭킹=페이북겜머니 주/월, 타이머 없음 | ❌ 신규 |
| 월간=방문횟수 | 월간=페이북겜머니(금액) | 🔧 수정 |
| 자유 연습(광고보고 한번더) | v0.10 강제 게이트만 존재 | 🔧 옵트인 추가 |
| 결과 정답공개 | 실종목명이 "최선의 한 수"에 혼재 | 🔧 프레이밍 강화 |
| 은근한 탐색 링크(#·상수) | 명령형 `cta-invest` | 🔧 초대형 링크로 교체 |
| 톤(버튼 최소화) | 결과창 버튼 다수 | 🔧 감축 |

## 2. 구현 항목

### A. 오늘의 종목 (신규)
- `game/daily.ts`: `getDailyScenarioIds(dateKey, manifest)` — 날짜 시드 해시로 **실데이터 2종** 결정(결정적).
- store(영속): `dailyDate`, `dailyDoneCount(0~2)`, `dailyScore`(오늘 손익 합).
- 홈: "오늘의 종목 N/2 · 자정까지 HH:MM:SS", 2판 완료 시 "오늘 완료 · 내일 다시 오기".
- 오늘의 종목 = **랭킹 반영**(ranked). 자정(날짜 변경) 시 리셋.

### B. 연속 출석 streak (신규)
- store(영속): `streak`, `lastPlayDay`. 플레이 시 어제→+1, 오늘→유지, 그 외→1(하루 거르면 끊김).
- 헤더/홈에 **🔥 N일 연속**. 증가 시 가벼운 축하 토스트.

### C. 랭킹 재편 (수정)
- `game/ranking.ts`: `getDailyRanking(dateKey)`(점수 mock) + `getMonthlyVisitRanking(monthKey)`(방문횟수 mock) — **서버 교체 지점을 순수 함수로 분리**.
- 화면 탭: **일간 / 월간**. 일간=오늘 점수·순위, 월간=이번달 방문횟수·순위. 상단 **자정 리셋까지 남은 시간** 타이머.
- 일간 점수 = 오늘 획득 페이북겜머니(손익 합). 월간 방문횟수 = 이번달 플레이 횟수.

### D. 자유 연습 모드 + 옵트인 (재설계, 강제 유지와 병행)
- 오늘의 종목 소진 후 홈/결과에 **옵트인** "연습 한 판 더(랭킹 미반영)" — 안 눌러도 무방.
- 기존 "3판마다 광고" 게이트는 유지(전체 플레이 카운트 기준). 연습판도 동일 규칙.

### E. 결과/복기 톤 개편 (수정)
- **리빌 프레이밍**: 상단 "이 차트는 사실 **[실제 종목명]**이었습니다" 명확화 + *내 판단 vs 실제 흐름* 1~2줄.
- **은근한 탐색 링크**: "이 종목, 실제로는 어떻게 됐을까? 👀" 1개(초대형). `constants.INVEST_LINK='#'` 상수 분리 → 추후 실제 투자정보 화면 교체.
- **버튼 감축**: 명령형 `cta-invest` 제거, 성향진단 축소. 하단은 "내일 다시 오기/처음으로" 위주. 투자 노출은 눈에 띄되 옵셔널.

## 3. 데이터 소스 분리(교체 가능)

```
game/ranking.ts    getDailyRanking(date) / getMonthlyVisitRanking(month)   // now mock → later fetch

```

## 4. 단계
1. constants·daily·ranking 골격 + store 상태(streak/daily/dailyScore/monthlyVisits)
2. 오늘의 종목 흐름 + 홈 카드/자정 타이머
3. streak 표시·연출
4. 랭킹 재편(일간 점수/월간 방문) + 타이머
5. 결과 톤 개편(리빌·초대형 링크·버튼 감축)
6. 연습 모드 옵트인
7. 빌드·헤드리스 검증 → 백업·배포

## 5. 비목표 / 원칙
- 강제 유도·강매·퍼널 배제(기존 광고 게이트는 결정에 따라 유지).
- 실제 서버·광고 SDK 미연동(모두 함수/상수로 분리하여 교체 지점만 제공).

## 0. TL;DR (경영 요약)

캔들런은 **실제 국내주식 과거 데이터로 캔들 차트 매매를 체험**하게 하는 모바일 웹 게임 프로토타입이다. "블라인드 종목을 보고 매수/관망/매도를 결정 → 결과를 복기"하는 루프가 이미 완성도 있게 동작하며, 지표·시그널·Tip·성과비교·판단평가 등 **학습 콘텐츠로서의 뼈대는 충분히 강하다.**

다만 현재는 **① 페이북 투자서비스로의 전환(활성화) 장치가 전무**하고, **② 다시 돌아올 이유(리텐션)와 공유(바이럴) 루프가 없으며**, **③ 3.7MB 단일 HTML·백엔드 부재·계측(Analytics) 부재로 운영·측정·확장이 불가능**하다. 또한 실제 종목명을 노출하는 금융 콘텐츠임에도 **규제(자본시장법·금융소비자보호법) 대응이 최소한**에 그친다.

본 PRD는 위 3대 갭을 해소하여 "재미있는 프로토타입"을 **"투자서비스 유입·전환을 실제로 만들어내고, 성과를 측정·개선할 수 있는 운영 가능한 콘텐츠"**로 끌어올리는 것을 목표로 한다. 3단계(MVP+ → 성장 → 확장) 로드맵과 P0~P2 요구사항, KPI, To-Be 아키텍처, 컴플라이언스 체크리스트를 정의한다.

## 1. 배경 및 목적

### 1.1 비즈니스 배경
- 페이북(BC카드) 내 **투자서비스 활성화**가 목표. 결제 앱 사용자에게 투자를 "어렵고 무서운 것"이 아니라 **가볍게 체험 가능한 것**으로 인지시키는 유입·전환 콘텐츠가 필요하다.
- 게임/시뮬레이션은 **낮은 진입장벽 + 학습효과 + 재미**를 동시에 제공하므로, 투자서비스 첫 경험(계좌개설·첫 거래) 유도에 적합한 포맷이다.

### 1.2 제품 목적
1. **인지·흥미 유발**: "차트 보는 게 재미있고 나도 할 수 있다"는 경험 제공
2. **학습**: 캔들·이동평균·시그널을 손실 리스크 없이 체득
3. **전환**: 게임 경험 → 실제 페이북 투자서비스 진입(활성화)으로 자연스럽게 연결
4. **측정·개선**: 유입~전환 퍼널을 계측하고 데이터 기반으로 개선

### 1.3 본 문서의 범위
- 포함: 현황 분석(As-Is), 갭 진단, 개선기회, 목표·KPI, 기능/비기능 요구사항, To-Be 아키텍처, 컴플라이언스, 로드맵, 측정계획
- 제외: 상세 UI 디자인 시안, API 상세 스펙 문서, 정식 법률 검토(별도 진행 필요)

## 2. 현황 분석 (As-Is)

### 2.1 기술 구조
| 항목 | 현재 상태 |
|---|---|
| 구성 | **단일 `index.html` (약 3.7MB)** — HTML/CSS/JS/데이터 모두 인라인 |
| 데이터 | 실제 국내주식 **300개 시나리오**를 `<script type="application/json">`에 임베드 (qbrick.db 출처, KOSPI/KOSDAQ 과거 일봉 OHLCV) |
| 렌더링 | Canvas 2D 직접 구현(캔들·지표·거래량·마커) |
| 백엔드 | **없음** (정적 파일). `./data/scenarios.json` fetch 폴백 코드는 있으나 미사용 |
| 상태 저장 | **없음** — 새로고침 시 지갑·진행 전부 초기화(지갑 10,000으로 리셋) |
| 계측 | **없음** — 이벤트/퍼널 추적 불가 |
| 빌드/테스트 | **없음** — 모듈화·테스트·CI 부재 |

### 2.2 핵심 게임 루프
1. **인트로**: 빠른 시작 / 직접 설정(난이도·봉수·지표 선택)
2. **플레이**: 블라인드 종목 차트 제시 → `매수/관망` 또는 `매도/보유` + `Tip` → 봉 예산(30/50/80) 소진까지 진행
3. **결과**: 내 수익률, Buy&Hold·코스피 대비, **최선의 한 수(복기)**, 판단 평가(좋았던 점/주의점), **실제 종목 공개**, 페이북머니 정산
4. **재도전**: "한 판 더" → 인트로

### 2.3 구현된 강점 (계승할 자산)
- ✅ **실데이터 기반 블라인드 학습** — look-ahead 차단(미래 봉 비공개) 설계가 견고함
- ✅ **시그널 점수 시스템** — MA/RSI/MACD/거래량 점수를 합산해 직관적 해석 제공
- ✅ **복기 중심 학습 설계** — "최선의 한 수", Buy&Hold·지수 대비, 판단 평가는 교육 콘텐츠로서 완성도 높음
- ✅ **모바일 퍼스트 UX** — 430px 기준, 한국식 캔들(상승 빨강/하락 파랑), 다크+골드 브랜딩
- ✅ **난이도/지표 커스터마이즈** — 초보/기본/고급, 지표 6종 토글

## 3. 갭 진단 (문제 정의)

### 3.1 🔴 P0 — 전환(활성화) 연결 부재 *(가장 치명적)*
- 게임이 끝나도 **페이북 투자서비스로 넘어가는 CTA·딥링크·유도 동선이 전혀 없다.** 결과화면은 "한 판 더"와 "처음으로"뿐.
- 결과에 기반한 **개인화 추천/성향 진단**이 없어 "그래서 나는 뭘 해야 하나"로 이어지지 않음.
- 페이북머니(가상재화)가 **실제 혜택/포인트/리워드와 연결되지 않아** 동기부여가 약함.
- **→ 활성화 목표를 위한 콘텐츠인데 활성화 전환 장치가 없음. 최우선 해결 과제.**

### 3.2 🔴 P0 — 리텐션·바이럴 루프 부재
- 단발성 원샷 게임. **다시 올 이유(일일 챌린지/미션/시즌/랭킹)가 없음.**
- **공유 기능 없음** — 결과를 자랑/공유할 카드가 없어 바이럴(무료 유입) 루프가 끊김.
- 진척(레벨/뱃지/누적 성과) 개념이 없어 성취·수집 동기 없음.

### 3.3 🔴 P0 — 운영·측정 불가 (계측·백엔드 부재)
- **Analytics 이벤트가 하나도 없어** 유입→플레이→완료→전환 퍼널을 측정할 수 없다. = 개선 의사결정 불가.
- 300 시나리오가 HTML에 박혀 있어 **콘텐츠 교체·추가·A/B가 배포 없이는 불가**. 초기 로드 3.7MB로 모바일 데이터/성능 부담.
- 상태 미영속 → 이탈 후 복귀 시 경험 단절.

### 3.4 🟠 P1 — 게임성/깊이 부족
- **공매도(숏) 진입 UI가 없음** — 코드에는 숏 마커/토스트가 있으나 컨트롤 버튼은 `매수/관망`뿐이라 **도달 불가능한 데드 UI**.
- **난이도가 Tip 횟수만 변경** — 초보/기본/고급이 지표·시나리오 난이도·시드 등 실질에 영향 없음(얕은 구현).
- 시나리오 선택이 **완전 랜덤** — 학습 목표별 큐레이션(교과서적 상승/하락/횡보/급등락 패턴) 없음.
- 봉수 모드 라벨 불일치(코드 토스트 "100봉 모드" vs 실제 30/50/80) 등 소소한 결함.

### 3.5 🟠 P1 — 학습 온보딩 부족
- 첫 사용자를 위한 **튜토리얼/가이드 투어가 없음.** 규칙 텍스트만 존재해 초보 이탈 위험.
- 시그널 점수의 산출 근거가 상대적으로 블랙박스 — "왜 이 점수인지" 심화 설명 부족.
- 체계적 커리큘럼(캔들 → 이평선 → 시그널 종합) 진행이 없음.

### 3.6 🟠 P1 — 규제·컴플라이언스 리스크
- **실제 종목명·기간을 결과에서 공개** → 특정 종목 매수·매도 추천으로 오인될 소지. 면책 문구는 있으나 **법률 검토 필요.**
- 과거 시세 데이터 **이용 라이선스(KRX/코스콤 등) 확인 필요.**
- 계좌 연동·리워드 도입 시 **개인정보·전자금융·금융소비자보호법** 적용 범위 확대.

### 3.7 🟡 P2 — 기술 품질/접근성
- 모놀리식 단일 파일 → 유지보수·협업·테스트 난이도 높음.
- 접근성(a11y): Canvas 위주라 스크린리더/저시력 대응 미흡, 색각 이상 대비 부족(빨강/파랑 의존).
- 로딩/에러/오프라인 상태 처리 없음. 사운드/햅틱 등 모바일 몰입요소 없음.

## 4. 목표 및 성공지표 (KPI)

### 4.1 North Star Metric
> **"게임 플레이를 통해 발생한 페이북 투자서비스 활성화 수"** (기간 내 게임 경험 후 투자서비스 진입/계좌개설/첫 거래로 이어진 사용자 수)

### 4.2 퍼널 & 목표지표
| 단계 | 지표 | 정의 | 초기 목표(예시) |
|---|---|---|---|
| 유입 | 진입 UV | 콘텐츠 진입 순사용자 | 측정 기반 확립 |
| 활성 | 게임 완료율 | 진입 대비 1판 이상 완료 | ≥ 60% |
| 몰입 | 재플레이율 | 완료자 중 2판 이상 | ≥ 40% |
| 리텐션 | D1/D7 복귀율 | 익일/7일 재방문 | D1 ≥ 25% |
| 바이럴 | 공유율 | 완료자 중 결과 공유 | ≥ 10% |
| **전환** | **투자서비스 CTA 클릭률** | 결과화면 CTA 클릭 | ≥ 15% |
| **전환** | **활성화 전환율** | CTA 클릭 → 실제 활성화 | KPI 핵심 |

### 4.3 학습 성과(보조)
- 판단 정확도 향상(라운드 반복에 따른 시그널 부합 매매 비율↑)
- Tip 사용 후 판단 개선율

## 6. 요구사항 (Requirements)

우선순위: **P0**(서비스화 필수) · **P1**(성장 핵심) · **P2**(고도화)

### 6.1 전환/활성화 (P0)
- **[P0] 결과화면 전환 CTA**: 게임 종료 후 "실제로 투자해보기" CTA → 페이북 투자서비스 딥링크(계좌개설/종목탐색/서비스 홈). 진입 지점·파라미터 추적 파라미터(UTM 등) 포함.
- **[P0] 성향/스타일 진단 결과**: 누적 매매 데이터로 투자 성향(안정형/균형형/공격형 등) 진단 카드 제공 → 성향별 서비스/콘텐츠 추천으로 연결.
- **[P1] 페이북머니 리워드 브릿지**: 가상 페이북머니 성과를 실제 혜택(포인트/쿠폰/이벤트 응모)과 연동(정책·법무 확인 후). 최소한 "리워드 안내 → 서비스 진입"으로 브릿지.
- **[P1] 컨텍스트 전환 유도**: "방금 본 지표를 실제 종목에서 확인하기" 등 학습→실전 연결 카피/링크.

### 6.2 리텐션/바이럴 (P0~P1)
- **[P0] 결과 공유 카드**: 수익률/등급/최선의 한 수 요약을 담은 공유 이미지(OG 카드/카카오 공유). 공유 링크에 유입 추적.
- **[P0] 상태 영속화**: 지갑·전적·진척을 로컬(비로그인) + 서버(로그인) 저장. 새로고침·재방문 연속성 확보.
- **[P1] 일일 챌린지/미션**: "오늘의 종목", 데일리 미션, 연속 출석(스트릭). 복귀 동기 제공.
- **[P1] 랭킹·리더보드**: 주간 수익률/정확도 랭킹(가상). 시즌제.
- **[P1] 진척 시스템**: 레벨/뱃지/누적 성과·타이틀 수집 요소.

### 6.3 게임성/학습 (P1)
- **[P1] 공매도(숏) UI 활성화** 또는 명시적 제거: 현재 데드 UI 정리. 숏 도입 시 초보 난이도에선 숨김.
- **[P1] 난이도 실질화**: 난이도별로 (a)지표 노출, (b)시나리오 변동성/패턴 난이도, (c)Tip·힌트, (d)수수료/슬리피지 반영 등을 차등.
- **[P1] 시나리오 큐레이션·태깅**: 학습목표별(상승추세/하락/횡보/골든크로스/과열조정 등) 태깅 후 커리큘럼·챌린지에 활용.
- **[P1] 온보딩 튜토리얼**: 첫 플레이 시 인터랙티브 가이드(캔들 읽기 → 매수/관망 → 결과 해석).
- **[P2] 학습 모드/설명 강화**: 시그널 점수 산출 근거 상세보기, 지표 학습 카드.
- **[P1] 거래 리얼리즘**: 수수료·세금·슬리피지 옵션(고급 난이도)으로 현실감·교육효과 강화.

### 6.4 데이터/콘텐츠 운영 (P0)
- **[P0] 시나리오 API/CDN 분리**: 데이터를 HTML에서 분리해 API 또는 CDN JSON으로 서빙. 지연 로딩·캐싱·버저닝. 초기 페이로드 대폭 축소.
- **[P0] 콘텐츠 관리(운영툴)**: 시나리오 추가/교체/비활성/태깅을 배포 없이 관리. A/B 실험 지원.
- **[P1] 데이터 파이프라인**: 원천(시세) → 시나리오 생성/검증 → 배포 자동화. 라이선스 준수 로깅.

### 6.5 계측/실험 (P0)
- **[P0] 이벤트 트래킹**: 진입/설정/매수/관망/매도/Tip/결과/CTA클릭/공유 등 전 이벤트 계측(GA4·Amplitude 등). 퍼널·리텐션 대시보드.
- **[P1] A/B 테스트 프레임워크**: CTA 카피·리워드·난이도 실험.

### 6.6 비기능 요구사항 (NFR)
- **[P0] 성능**: 모바일 초기 로드 목표(예: 초기 JS < 300KB, LCP < 2.5s). 데이터 지연 로딩.
- **[P0] 안정성**: 로딩/에러/오프라인/빈 데이터 상태 처리.
- **[P1] 접근성**: 색각 대비(빨강/파랑 외 보조 표기), 명암비, 주요 동작 키보드·스크린리더 대응, 탭 타깃 크기.
- **[P1] 보안/개인정보**: 로그인·계좌 연동 시 최소수집·암호화·동의 관리.
- **[P1] 유지보수성**: 모듈화(빌드 도입), 핵심 로직 단위테스트, CI.

## 13. v0.4 구현 반영

v0.4에서 프로토타입(v0.3, 단일 `index.html`)을 **React + TypeScript + Vite 앱(`app/`)으로 재구축**하고 데이터를 분리했다. 아래는 요구사항별 구현 결과다.

### 13.1 데이터 ↔ 앱 분리 (P0 · 완료)
- 300개 실데이터가 HTML에 임베드되던 구조를 제거하고, **`app/public/data/`로 분리**.
  - `manifest.json` — 전 시나리오 메타(시장·섹터·패턴·난이도·기간·소스)만 담은 경량 인덱스(≈137KB).
  - `scenarios/<id>.json` — 라운드에서 실제 필요한 시나리오만 **지연 로드**(개별 ≈12KB).
- 초기 페이로드: **3.7MB 단일 파일 → JS 181KB(gzip 60KB) + 매니페스트**. 콘텐츠 교체·증량이 앱 배포와 분리됨.
- 파이프라인: `scripts/build_data.py`(실데이터 추출 + 랜덤 생성 → 분리 저장).

### 13.2 랜덤 생성 + KOSPI/KOSDAQ 구분 + 증량/최근 데이터 (P0 · 완료)
- **총 570개 시나리오** = 실데이터 300(KOSPI) + 랜덤 생성 270(KOSPI 120 / KOSDAQ 150).
- 시장분포 **KOSPI 420 / KOSDAQ 150**, **2020년 이후 330개**. 각 시나리오에 `market` 태그를 부여해 플레이·결과 화면에 KOSPI/KOSDAQ를 표시하고, 설정에서 시장을 선택.
- 랜덤 생성 시나리오는 패턴(상승/하락/횡보/변동)·난이도를 자동 태깅하고 **2020년 이후 날짜**로 부여. KOSDAQ은 변동성을 높여 시장 특성을 반영.
- ⚠️ **한계/투명성**: 실행 환경의 이그레스 제한으로 실시간 신규 시세(KRX/네이버/야후 등) 수집이 불가하여 KOSDAQ·증량분은 **합성 데이터**로 생성했다. 앱은 결과 화면에서 실데이터는 실제 종목명, 합성은 **"연습용 합성 시나리오"**로 명시해 오인을 방지한다. 실서비스에서는 시세 라이선스 확보 후 실데이터로 대체 필요(§9, §12).

### 13.3 반응형 동적 UI (P1 · 완료)
- 고정 430px 레이아웃 → **유동 셸(모바일 460px, 데스크톱 620px)** + 차트가 `ResizeObserver`로 컨테이너 크기에 맞춰 실시간 리사이즈. 화면 폭에 따라 표시 봉 수가 자동 조절(30~90봉).
- 검증: 뷰포트 390px → 데스크톱 1200px 전환 시 차트 340px→570px로 동적 확장 확인.

### 13.4 명확한 게임 플로우 (P0 · 완료)
- **설정 → 실행 → 성과분석 → 지속/종료** 흐름을 스토어(zustand) 기반 화면 전환으로 구현.
  - **설정**: 난이도·시장·봉수·최근데이터 우선·지표 선택 + 빠른 시작.
  - **실행**: 블라인드 차트·시그널·포지션·Tip·매수/공매도/관망/청산.
  - **성과분석**: 내 매매 vs Buy&Hold vs 벤치마크(추정) vs 구간 최선 포착률, 최선의 한 수 복기, 판단 평가, 실제 시나리오 공개, **투자 성향 진단**.
  - **지속/종료**: `한 판 더` / `설정 변경` / `공유`.

### 13.5 완성도·전환 요소 (P0~P1 · 부분 완료)
- **투자 성향 진단**(안정형/균형형/공격형/학습형) → 성향별 전환 CTA.
- **전환 CTA**: 결과 화면에 투자서비스 이동 지점 배치(현재 데모 훅 — 실제 딥링크 연동은 §12 의사결정 필요).
- **결과 공유**: `navigator.share`/클립보드 기반 요약 공유.
- **영속화**: 페이북머니·누적 전적(라운드/승률/성향)을 localStorage에 저장.
- **공매도 실연결**: v0.3의 데드 UI를 고급 난이도에서 실제 진입 가능하도록 연결.

### 13.6 v0.4에서 미포함(향후) 
계측(Analytics)·A/B·백엔드(서버 전적/랭킹)·일일 챌린지/시즌·실제 투자서비스 딥링크·실데이터 라이선스 연동·접근성 고도화는 후속 단계(로드맵 Phase 2~3)로 남긴다.
